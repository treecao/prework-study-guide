HTML

  • The head element contains information about the webpage.
  • The body element represents the visible content shown to the user.
  • The header size is determine by h and the corresponding number size. 1 being the largest, 6 being the smallest.
  • The attributes id and class are similar but we have not yet learned their differences.
  • HTML needs to be written in sequential order.
  • Open with Live Server is not an option for me, but I use open with default browser for the same effect.

CSS

  • A margin indicates how much space we want around the outisde of an element.
  • A padding indicates how much space we want around the content inside an element.
  • Three ways to style: inline CSS, internal CSS Style Sheet, and external CSS Style Sheet
  • Inline CSS: style an element directly in the HTML file
  • Internal Style Sheet contains CSS rules for the webpage in the head section of the HTML file
  • External CSS Style Sheet is most common. Preferred because it allows devs to keep all their CS rules in separate files
  • Declarations (rules) consist of two components: the CSS property we want to apply and the value of the property
  • Do Not Repeat Yourself (DRY) - a coding principle that encourages devs to reuse or share pieces of code in order to reduce the number of lines of code that need to be written or maintained
  • Class selector starts with a period (.)

Git

  • Git is fun
  • Add new branch: git checkout -b feature/new name
  • Double-check lock branch is synced to GH: git pull origin main
  • Push changes: git push origin branch name here
  • Switch to main branch: git checkout main

JavaScript

  • A variable is a named container that allows us to store data in our code.
  • Control flow is the order in which a computer executes code in a script.
  • Variables may hold different data types: String, Number, Boolean, Array, and Object.
  • Operator is a math sybol that produces a result based on two values or variables.
  • Conditionals are code structures used to test if an expression returns true or not.
  • Functions package functionality you wish to reuse.
  • Arguments: bits of data functions need to do their job
  • Events are code structures that listen for activity in the browser and run code in response.
  • Symbols are sometimes called "atoms"
  • Array: single variable used to hold group of data
  • Index: unit number within an array
  • JS arrays are zero-indexed & sequential (1st item is index of 0, not 1).
